Proposed Agent Architecture

◆ Classification Agent
Determines file type (File 1/2/3). Cascades: heuristic → LLM → vision model if uncertain. Delegates to Verification Agent when confidence < 0.7.
◆ Extraction Agent
Extracts structured data from documents. File 1: deterministic (pdfplumber). File 2: OCR + LLM. Can be invoked with different strategies on retry.
◆ Verification Agent
Scores confidence per extracted field (0–1). Compares extraction against source page image. Routes low-confidence fields to Retry Agent or Human.
◆ Retry Agent
Re-attempts extraction with alternative strategy: different prompt, page-by-page mode, or vision model. Max 2 retries per packet. Enforces cost budget.
◆ Reconciliation Agent
Runs deterministic rules (math, cross-file, retainage, continuity). NOT LLM-powered — remains rule-based. Delegates ambiguous cases to human.

Confidence-Based Routing Logic

Score ≥ 0.85 High confidence — auto-approve field. No human review needed. AUTO-APPROVE
Score 0.50 – 0.84 Medium confidence — flag for quick human spot-check. Show extracted value + source highlight. SPOT-CHECK
Score < 0.50 Low confidence — delegate to Retry Agent with alternative strategy. If retry also low, escalate to human. RETRY → ESCALATE
Classification uncertain Heuristic confidence < 0.7 — cascade to LLM classifier, then vision model if still uncertain. CASCADE

Implementation Approach Comparison

Raw Custom Code

  • if/else routing in pipeline.js
  • No abstraction for agents
  • Fast to start, messy at scale
  • Hard to add new agents later
  • Debugging: grep through logs
  • ≤3 agents max before spaghetti

Lightweight Custom Orchestrator

  • ~300 lines structured routing
  • Consistent agent interface
  • Confidence threshold config
  • Full delegation logging
  • Cost budget enforcement
  • Easy to add agents (5-10)
  • No external dependency

Full Framework (SK / LangGraph)

  • Heavy dependency
  • Learning curve for team
  • Abstractions for 10+ agents
  • Built-in memory/state
  • Overkill for 5 agents
  • Debugging through framework layers
  • Migration: 12-16 weeks

Key Risks & Mitigations

HIGH
Verification agent disagrees with correct extraction
Mitigation: Verification scores confidence, never auto-rejects. Only flags or retries. Human is final arbiter.
HIGH
Latency doubles/triples on complex packages
Mitigation: Acceptable if human review wait-time drops proportionally. Track total end-to-end time including human queue.
MED
Cost blowup on messy PDFs (repeated retries)
Mitigation: Per-package token budget cap. Max 2 retries per packet. After budget exceeded → escalate to human.
MED
Debugging multi-agent chains becomes opaque
Mitigation: Every agent call logs input hash, output, confidence score, and delegation decision. Full trace per packet.
LOW
Infinite retry loops
Mitigation: Hard cap of 2 retries. Circuit breaker pattern — if retry doesn't improve score by ≥0.15, stop immediately.

VERDICT: FEASIBLE — Implement Selectively

Multi-agent delegation is technically feasible and will measurably reduce human review burden. Build a lightweight custom orchestrator (not a vendor framework) with Verification Agent first, measure accuracy lift, then expand. Keep all deterministic logic (G703, reconciliation, state machine) untouched. A full agent framework (Semantic Kernel, LangGraph) is overkill for 4–5 agents with fixed routing.

Revised Recommendation: LangGraph-Based Architecture

REVISED DECISION: Adopt LangGraph as Orchestration Framework

The system needs dynamic planning, extensible agent registration, and proper decision-making for a growing business application. A lightweight custom orchestrator cannot scale to handle unknown future scenarios. LangGraph provides state-machine orchestration with native human-in-the-loop, checkpointing, and tool abstraction — purpose-built for multi-step document processing.

LangGraph State Schema

Python — Graph State Definition
from typing import TypedDict, Annotated, Literal
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.postgres import PostgresSaver

class PackageState(TypedDict):
    # Core identifiers
    package_id: int
    project_id: int
    
    # Documents
    documents: list[dict]           # [{blob_url, filename, page_count, classification, confidence}]
    page_images: dict[str, str]     # {page_key: image_url}
    
    # Classification
    classifications: dict[str, dict]  # {doc_id: {file_type, confidence, method_used}}
    classification_confirmed: bool
    
    # Plan
    extraction_plan: dict           # {subcontractors: [{name, sov_lines, confirmed}]}
    plan_confirmed: bool
    
    # Extraction results
    file1_data: dict                # {gc_header: {...}, sov_lines: [...]}
    file2_data: dict                # {sub_headers: [...], sub_sov_lines: [...]}
    file3_data: dict                # {compliance_docs: [...]}
    
    # Verification
    confidence_scores: dict         # {field_path: {score, evidence_page, bbox}}
    fields_auto_approved: list[str]
    fields_flagged: list[str]
    fields_for_retry: list[str]
    
    # Retry tracking
    retry_count: int
    retry_budget_remaining: int     # tokens remaining in retry budget
    
    # Reconciliation
    exceptions: list[dict]          # [{type, severity, description, evidence}]
    reconciliation_result: dict
    
    # Human decisions
    human_decisions: dict           # {field_path: {action, override_value, reason}}
    
    # Audit
    agent_trace: list[dict]         # [{agent, action, timestamp, tokens_used, result}]
    total_tokens: int
    total_cost: float

# Build the graph
graph = StateGraph(PackageState)

# Add nodes
graph.add_node("ingest", ingest_documents)
graph.add_node("classify", classify_documents)
graph.add_node("confirm_classification", confirm_classification)
graph.add_node("extract_file1", extract_file1)
graph.add_node("generate_plan", generate_plan)
graph.add_node("confirm_plan", confirm_plan)
graph.add_node("extract_file2", extract_file2)
graph.add_node("extract_file3", extract_file3)
graph.add_node("verify", verify_extraction)
graph.add_node("retry", retry_extraction)
graph.add_node("reconcile", reconcile)
graph.add_node("generate_exceptions", generate_exceptions)
graph.add_node("human_review", human_review)
graph.add_node("finalize", finalize_package)

# Add edges
graph.add_edge("ingest", "classify")
graph.add_conditional_edges("classify", route_classification)
graph.add_edge("confirm_classification", "extract_file1")
graph.add_edge("extract_file1", "generate_plan")
graph.add_conditional_edges("generate_plan", route_plan)
graph.add_edge("confirm_plan", "extract_file2")
graph.add_edge("extract_file2", "extract_file3")
graph.add_edge("extract_file3", "verify")
graph.add_conditional_edges("verify", route_by_confidence)
graph.add_edge("retry", "verify")  # retry loops back
graph.add_edge("reconcile", "generate_exceptions")
graph.add_edge("generate_exceptions", "human_review")
graph.add_edge("human_review", "finalize")
graph.add_edge("finalize", END)

# Compile with checkpointing
checkpointer = PostgresSaver.from_conn_string(DATABASE_URL)
app = graph.compile(checkpointer=checkpointer)
