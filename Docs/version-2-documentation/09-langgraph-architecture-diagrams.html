✅ AIA Standards: G702 Application for Payment, G703 Continuation Sheet, sequential app numbering
✅ Math Validation: Contract Sum = Original + Net COs, Payment Due = Earned - Previous - Retainage
✅ Retainage Rules: State-specific (typically 10% until 50% complete, then 5%), trade-specific exceptions
✅ Stored Materials: Contract allowance check, photo/receipt verification, separate retainage tracking
✅ Change Order Sequencing: COs must be approved before pay app date, cumulative tracking
✅ Cross-Document Reconciliation: G702 totals = G703 sum, Sub invoices = G703 allocations
✅ Lien Law Compliance: State-specific requirements, conditional waivers, preliminary notices
✅ Anomaly Detection: Compare to previous apps, flag unusual velocity, detect duplicates

14. Architecture Decision Summary

✅ What Makes This a True Multi-Agent System:

1. Supervisor Pattern: Orchestrator coordinates specialist agents, not a flat peer-to-peer network
2. Agent Specialization: Each agent has deep expertise in one domain (extract, verify, reconcile)
3. Self-Validation: Agents check their own work and retry until quality thresholds met
4. Dynamic Routing: Orchestrator uses LLM reasoning to decide which agent to invoke when
5. Human-in-the-Loop: Seamless escalation when agents need human judgment
6. Shared State: LangGraph manages PayAppState across all agents and nodes
7. Auditability: Complete log of agent decisions, user responses, and workflow state changes

🎯 When to Use Agents vs Deterministic Nodes:

Use Agents (ReAct Pattern) When:
  • Task requires multi-step reasoning (Extraction: try strategy A, if fails try B)
  • Unknown number of iterations needed (keep extracting until confidence >85%)
  • Need to choose between multiple tools dynamically (vision vs OCR vs Document Intelligence)
  • Task complexity varies by input (some documents easy, some need multiple retries)
Use Deterministic Nodes When:
  • Fixed algorithmic logic (PDF → convert to images, always same process)
  • Simple classification (3-tier cascade: heuristics → text LLM → vision LLM)
  • Math validation (check if A + B = C, deterministic Python)
  • Data formatting (prepare JSON for UI, no reasoning needed)

    """Quality Agent detects Tier 2 FLAG items"""

20. Human-in-the-Loop - Indefinite Pause Pattern

💡 Key Insight: With only 1 app/month, there's no need for timeout logic. User can take days to respond without impacting other workflows. Indefinite pause is the simplest, most user-friendly approach for low-volume processing.

from langgraph.graph import StateGraph
from langgraph.checkpoint.postgres import PostgresSaver

# Configure graph with interrupt
graph = StateGraph(PayAppState)
graph.add_node("manager", manager_node)
graph.add_node("extraction", extraction_node)
graph.add_node("quality", quality_node)
graph.add_node("human_review", human_review_node)  # Interrupt point
graph.add_node("reconciliation", reconciliation_node)

# Add conditional edge: pause if human input needed
graph.add_conditional_edges(
    "manager",
    should_ask_user,  # Returns "human_review" or "continue"
    {
        "human_review": "human_review",  # Goes to interrupt
        "continue": "reconciliation"
    }
)

# Compile with interrupt BEFORE human_review
app = graph.compile(
    checkpointer=PostgresSaver.from_conn_string(postgres_url),
    interrupt_before=["human_review"]  # ⏸️ Pauses here
)

# First invocation - will pause at human_review
config = {"configurable": {"thread_id": "app_july_2026"}}
result = app.invoke(initial_state, config)

# Returns: {"status": "PENDING_HUMAN_INPUT", "question": "Are these the same contractor?"}
# Workflow is now paused, state saved to PostgreSQL

# ... User can close browser, come back hours/days later ...

# User answers question
state_with_answer = result["state"]
state_with_answer["user_responses"]["contractor_match"] = "yes_same"

# Second invocation - resumes from human_review checkpoint
final_result = app.invoke(state_with_answer, config)

# Workflow continues: reconciliation → finalization → complete!
